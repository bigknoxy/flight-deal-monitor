📚 Flight Deal Monitor Documentation

Installation Instructions

Clone Repository

git clone https://github.com/bigknoxy/flight-deal-monitor.git cd flight-deal-monitor

Install Dependencies

pip install -r requirements.txt

Or with pipx (recommended for fli):

pip install pipx pipx install fli

Configure Environment

cp config/.env.example .env # Edit .env with your credentials
Required Variables:
  • SEARCHAPI_API_KEY - Get from searchapi.io
  • TELEGRAM_BOT_TOKEN - Create via @BotFather
  • TELEGRAM_CHAT_ID - Your Telegram chat ID

Start Application

python -m app.main

Application will be available at http://localhost:8000

Configuration Guide

Environment Variables (.env)

Variable Description Required Default
SEARCHAPI_API_KEY Google Flights API key Yes (for reliable results) -
TELEGRAM_BOT_TOKEN Telegram bot token Yes -
TELEGRAM_CHAT_ID Telegram chat/group ID Yes -
DUFFEL_API_TOKEN Duffel API backup token Optional -
DATABASE_URL Database connection string No sqlite:///./flight_deals.db
LOG_LEVEL Logging verbosity No INFO

App Configuration (config/app.yaml)

Parameter Description Default
home_airports List of departure airports (IATA codes) ["MCI"]
destinations List of destination airports (IATA codes) Multiple international airports
mistake_fare_threshold ≥70% below median = mistake fare 0.70
flash_sale_threshold ≥50% below median = flash sale 0.50
cache_ttl_minutes Price cache TTL in minutes 360 (6 hours)
regular_sweep_interval Regular sweep interval in seconds 1800 (30 min)
mistake_sweep_interval Mistake fare sweep interval in seconds 900 (15 min)
Note: The fli library works intermittently as Google actively blocks repeated API requests. SearchAPI is recommended as the reliable fallback.

API Usage

fli Library (Primary)

The fli library provides FREE access to Google Flights data:

# No API key required - uses curl_cffi from fli.models import Airport, FlightSearchFilters, FlightSegment, PassengerInfo from fli.search import SearchFlights searcher = SearchFlights() segments = [FlightSegment( departure_airport=[[Airport.LHR, 0]], arrival_airport=[[Airport.JFK, 0]], travel_date="2026-07-15", )] filters = FlightSearchFilters( passenger_info=PassengerInfo(adults=1), flight_segments=segments, ) results = searcher.search(filters, top_n=10)

SearchAPI (Fallback)

Reliable Google Flights API at $4/1000 requests:

import httpx API_KEY = "your_key" params = { "engine": "google_flights", "departure_id": "MCI", "arrival_id": "LHR", "outbound_date": "2026-07-15", "api_key": API_KEY, } resp = httpx.get("https://www.searchapi.io/api/v1/search", params=params) flights = resp.json().get("best_flights", [])

Duffel API (Backup)

Air booking API with offer requests:

import httpx headers = { "Authorization": f"Bearer {api_token}", "Duffel-Version": "v1", "Content-Type": "application/json", } data = { "data": { "slices": [{ "origin": "MCI", "destination": "LHR", "departure_date": "2026-07-15", }], "passengers": [{"type": "adult"}], "max_offers": 10, } }

Deployment Options

Docker Compose (Recommended)

docker-compose up -d
  • Multi-stage build optimized for production
  • Auto-restart on failure
  • Health check built-in

Docker (Manual)

docker build -t flight-deal-monitor . docker run -d \ --name flight-monitor \ --restart unless-stopped \ -p 8000:8000 \ --env-file .env \ flight-deal-monitor

Kubernetes
