Primary: fli (FREE Google Flights via curl_cffi)
Fallback: SearchAPI ($4/1K requests)
Backup: Duffel Air Booking API

API Selection Logic

  • fli - First choice, FREE access to Google Flights data
  • SearchAPI - Used when fli fails (most reliable fallback)
  • Duffel - Final backup option
  • All clients return normalized flight data for consistent processing

Key Components

scheduler_jobs.py

  • run_regular_sweep() - Weekly price checks for all configured routes
  • run_mistake_sweep() - Daily deep checks for popular high-volume routes
  • _scan_route() - Core scanning logic with API failover

cache.py

  • PriceCache with 6-hour TTL (360 minutes)
  • Reduces API costs by skipping stable price searches
  • Stores flight results keyed by route + date

alert.py

  • TelegramBot class for sending notifications
  • Rate limiting (max 10 alerts per hour by default)
  • Markdown-formatted messages with booking links

Database Schema

FlightDeal: - route_id: str (SHA256, 16 chars) - origin: str (IATA code) - destination: str (IATA code) - departure_date: date - airline: str - flight_numbers: str - original_price_usd: float (median) - current_price_usd: float - price_drop_percent: float - deal_type: str (flash_sale/mistake_fare) - booking_url: str - seen_at: datetime AlertHistory: - flight_deal_id: int (FK) - telegram_message_id: str - status: str (sent/failed) - error_message: Optional[str] - created_at: datetime JobRun: - job_id: str (regular_sweep/mistake_sweep) - started_at: datetime - completed_at: datetime - status: str (success/failed) - deals_detected: int - alerts_sent: int - error_message: Optional[str]
